## 1. 系統概述

本系統為一套基於 Python + Streamlit 建構的量化交易回測平台，支援台股與虛擬貨幣的策略回測、多策略比較、財報分析等功能。系統採用 SQLite 作為本地資料快取，透過 yfinance 抓取台股歷史股價，透過 ccxt 抓取幣安虛擬幣行情，並整合 FinMind API 提供財報基本面資料。

### 系統目標

- 提供個人投資者快速驗證交易策略的工具
- 支援多股票、多策略並行比較，縮短回測決策時間
- 透過本地資料庫快取降低 API 呼叫頻率
- 提供直觀的圖表介面，無需程式背景即可操作

## 3. 模組說明

### 3.1 `database.py` — 資料存取層

負責所有 SQLite 資料庫的讀寫操作。

| 函式 | 說明 |
|------|------|
| `init_db()` | 建立 `stock_price` 資料表（若不存在） |
| `save_stock_prices(df, stock_code)` | 儲存股票歷史價格，主鍵衝突自動覆蓋 |
| `load_stock_prices(stock_code, start_date, end_date)` | 讀取指定股票與日期區間的價格資料 |
| `delete_stock_prices(stock_code)` | 刪除指定股票的所有快取資料 |
| `get_latest_date(stock_code)` | 查詢資料庫中該股票最新的資料日期 |
| `save_strategy_result(...)` | 儲存回測結果 |
| `load_strategy_result(...)` | 讀取回測結果 |

**重要設計決策**：yfinance 下載時使用 `auto_adjust=False`，保留原始市價（Open/High/Low/Close），避免還原除權息後股價與看盤軟體數值不符。

### 3.2 `strategy.py` — 策略邏輯核心

#### 股票清單載入

從 `stocks.db` 的 `stock_list` 資料表讀取上市股票代號與名稱，建立供各頁面使用的 `stock_list` 字典（格式：`{"2330.TW": "台積電", ...}`）。

#### 持倉狀態機 `_build_position(buy, sell)`

所有策略共用的持倉邏輯，採用狀態機取代 `ffill`，確保回測正確性：

```
規則：
  - buy 訊號出現  → position = 1（進場，持有直到 sell）
  - sell 訊號出現 → position = 0（出場，空手）
  - 兩者同時出現  → sell 優先（保守原則）
  - 初始狀態      → position = 0（空手）
```

> **為何不用 `ffill`**：`ffill` 會將無訊號日（`Position = 0`）填為前一個訊號值，導致策略幾乎永遠持倉，累積報酬在末端出現異常暴衝或暴跌。

#### `apply_strategy(df, strategy_name, params)` → `df`

輸入股價 DataFrame 與策略名稱及參數，輸出附帶 `Position` 欄位的 DataFrame。

### 4.2 回測系統 `2_回測系統.py`

**功能**：單一股票 × 單一策略的完整回測，含蠟燭圖、績效圖、績效總表。

#### 輸入參數

| 參數 | 元件 | 說明 |
|------|------|------|
| 股票選擇 | Selectbox | 從 stock_list 選擇 |
| 開始 / 結束日期 | Date input | 回測區間 |
| 策略選擇 | Selectbox | 8 種策略 |
| 策略參數 | Slider / Number input | 依策略動態產生 |

#### 輸出結果

| 輸出 | 說明 |
|------|------|
| 蠟燭圖 | 原始市價（非還原價） |
| 累積報酬率比較圖 | 策略 vs 買入持有 |
| 策略績效總表 | 期間、報酬率、風險、最大回撤 |
| 最新交易訊號 | 空手 / 持有（買入）/ 放空 |
| 夏普比率 | 年化，以 240 交易日計算 |

#### 快取機制

- 資料庫有資料 → 直接讀取，不重打 API
- 資料庫無資料 → 從 yfinance 下載並存入資料庫
- 提供「🗑️ 清除此股票快取並重新下載」按鈕，強制重新下載

#### 資料清理流程

```
1. sort_index()              — 確保日期遞增
2. pd.to_numeric(errors='coerce') — Close 強制轉數字
3. 移除 Close 為 NaN 的列
4. pct_change() 計算日報酬率
5. 過濾 |DailyReturn| >= 0.5  — 排除單日漲跌超過 50% 的異常資料
```

### 4.3 策略比較 `3_策略比較.py`

**功能**：多股票 × 多策略並行回測，產出比較表與長條圖。

#### 輸入參數

| 參數 | 元件 |
|------|------|
| 股票選擇（多選） | Multiselect |
| 策略選擇（多選） | Multiselect |
| 開始 / 結束日期 | Date input |

#### 輸出結果

- 績效比較表（累積報酬率、夏普比率、最大回撤）
- 各股票各策略累積報酬率長條圖

#### 績效指標計算

```python
# 累積報酬率（幾何）
cum_return = (1 + strategy_returns).cumprod().iloc[-1] - 1

# 夏普比率（台股 240 交易日）
sharpe = (mean / std) * sqrt(240)

# 最大回撤
mdd = ((cum - cum.cummax()) / cum.cummax()).min()
```

#### 偏好記憶

上次選擇的股票與策略會儲存至 `user_selection.json`，下次開啟自動帶入。

### 4.5 財報比較 `5_財報比較.py`

**功能**：透過 FinMind API 抓取台股財報資料，比較多支股票近3年的平均財務指標。

#### FinMind API 規格

| 項目 | 說明 |
|------|------|
| 資料集 | `TaiwanStockFinancialStatements`（綜合損益表） |
| 回傳格式 | Long format（`date`, `stock_id`, `type`, `value`） |
| 季報代號 | `"Q"` |
| 年報代號 | `"A"` |
| 股票代號格式 | 純數字（`2330`），需去除 `.TW` 後綴 |
| 每日額度 | 免費帳號約 600 次/hr |

#### Token 登入機制

- 左側 Sidebar 提供密碼輸入框
- Token 僅存於 Session，不寫入磁碟
- 未輸入 Token 時整頁停止，不消耗 API 額度
- `@st.cache_resource` 快取登入狀態
- `@st.cache_data(ttl=3600)` 快取財報資料一小時

#### 財報科目對應

| 指標 | 主要科目名稱 | 備用科目名稱 |
|------|------------|------------|
| 營收 | 營業收入 | 營業收入合計、收益合計 |
| 營業利益 | 營業利益（損失） | 營業利益、營業損益 |
| 稅後淨利 | 本期淨利（淨損） | 本期淨利、稅後淨利、本期損益 |

> 注意：不同公司（尤其金融股）科目名稱略有不同，系統提供除錯 expander 顯示原始科目清單。

### 4.6 虛擬幣回測 `6_虛擬幣回測.py`

**功能**：透過 ccxt 連接 Binance，支援多交易對 × 多策略的虛擬幣回測，並產出彙總比較圖表。

#### 支援交易對

| 計價幣 | 交易對 |
|--------|--------|
| USDT | BTC、ETH、XRP、ADA、LINK、VET、DOGE |
| BTC | ETH、XRP、ADA、LINK、VET、DOGE |

BTC 計價交易對會自動乘以 BTC/USDT 匯率換算為 USDT 基準，方便跨對比較。

#### K 線週期

| 選項 | ccxt 參數 |
|------|-----------|
| 日線 | `1d` |
| 4 小時線 | `4h` |
| 1 小時線 | `1h` |
| 30 分鐘線 | `30m` |

#### 虛擬幣專屬策略

除支援所有台股策略外，額外提供：

**SMA/Hull 趨勢策略**
- 類型：`sma`（簡單移動平均）或 `hull`（Hull 移動平均）
- 參數：`n1`（短期週期，預設 30）、`n2`（長期週期，預設 130）
- 邏輯：短期均線上穿長期均線買入，下穿出場

#### 輸出圖表

| 圖表 | 說明 |
|------|------|
| 個別績效圖 | 每個交易對 × 策略組合的買入持有 vs 策略累積報酬率 |
| 彙總折線圖 | 所有組合的累積報酬率曲線疊加比較 |
| 彙總長條圖 | 各交易對 × 各策略最終報酬率並排 |
| 績效總表 | 報酬率、年化波動、最大回撤、夏普比率、最新訊號 |

**年化基準**：虛擬幣市場全年 365 天交易，夏普比率以 `sqrt(365)` 計算。

## 5. 策略規格

### 5.1 策略清單

| 策略名稱 | 買入條件 | 賣出條件 | 主要參數 |
|---------|---------|---------|---------|
| 簡單均線交叉 | 短期 SMA 上穿長期 SMA | 短期 SMA 下穿長期 SMA | 短期均線、長期均線 |
| 反轉策略 | N 日跌幅 ≥ 指定閾值 | N 日跌幅 < 指定閾值 | 觀察天數、跌幅閾值 |
| 突破策略 | 收盤價突破 N 日高點 | 收盤價跌破 N 日低點 | 突破天數 |
| RSI 策略 | RSI 由低於閾值回升穿越 | RSI 高於賣出閾值 | RSI 期間、買入閾值、賣出閾值 |
| MACD 策略 | MACD 線上穿訊號線 | MACD 線下穿訊號線 | 短期 EMA、長期 EMA、訊號線 |
| 布林通道策略 | 收盤價跌破下軌 | 收盤價突破上軌 | 期間、標準差倍數 |
| 黃金交叉 EMA 策略 | 短期 EMA 上穿長期 EMA | 短期 EMA 下穿長期 EMA | 短期 EMA、長期 EMA |
| 唐奇安通道策略 | 收盤價突破 N 日最高價 | 收盤價跌破 N 日最低價 | 期間 |
| SMA/Hull 趨勢策略 | 短趨勢線上穿長趨勢線 | 短趨勢線下穿長趨勢線 | type、n1、n2 |

### 5.2 持倉值定義

| Position 值 | 意義 |
|------------|------|
| `1` | 持有（多單） |
| `0` | 空手（無部位） |
| `-1` | 放空（本系統目前不主動做空，僅作為訊號標記） |

### 5.3 報酬率計算

```python
df['DailyReturn'] = df['Close'].pct_change()
df['Strategy']    = df['Position'].shift(1) * df['DailyReturn']

# shift(1) 的原因：今日收盤產生訊號，明日開盤才能執行，
# 使用前一日的 Position 乘以今日報酬率，避免未來函數偏差
```

## 8. 績效指標定義

| 指標 | 公式 | 說明 |
|------|------|------|
| 累積報酬率 | `(1 + r).cumprod() - 1` | 幾何累積，非算術加總 |
| 夏普比率（台股） | `mean(r) / std(r) × √240` | 240 個交易日年化 |
| 夏普比率（虛擬幣） | `mean(r) / std(r) × √365` | 365 天年化 |
| 年化波動率 | `std(r) × √240` 或 `× √365` | 依市場別調整 |
| 最大回撤（MDD） | `min((cum - cum.cummax()) / cum.cummax())` | 策略淨值計算 |

## 9. 已知限制與注意事項

### 資料面

- yfinance 資料不包含盤中即時報價，最快 T+1 更新
- FinMind 免費帳號每小時限制 600 次請求，大量股票財報查詢建議分批執行
- 部分小型股或 ETF 在 FinMind 財報科目名稱可能與標準不符，需透過除錯 expander 確認

### 回測面

- 本系統為**收盤價回測**，不考慮盤中滑價、手續費、稅金
- 未模擬部位大小（資金管理），所有訊號視為全倉操作
- 不支援當沖策略（最小週期為日線）
- 台股漲跌停限制（約 ±10%）未在回測中強制執行
- 虛擬幣回測未考慮交易所手續費（Binance 現貨約 0.1%）

### 系統面

- 資料庫為本地 SQLite，不支援多人同時寫入
- 若資料庫已有舊資料（調整價格），需手動點擊「清除快取」重新下載
- `stocks.db` 需至少執行一次「更新股票清單」頁面才能正常使用其他功能
